Layout rule (general — applies to any sub-merge node taking N sibling inputs):

  • Each input gets its own row.
  • Input i's LEFT edge sits at i · STAIR_FRAC · L from the merge's left edge (default STAIR_FRAC = 0.6).
  • Merge tile's width = L · (1 + STAIR_FRAC · (N − 1)) — exactly spans from input 0's left edge to input N−1's right edge.
  • Tap → merge: straight vertical line from input's bottom-mid down to merge's top edge. No curve.
  • Sequential between inputs: right-mid of input i → top-mid of input i+1, smooth right-angle bezier (out, then down).
